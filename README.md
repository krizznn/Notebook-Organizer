## Описание

**Notebook Organizer** — консольное приложение для управления заметками, написанное на **Python 3**.  
Поддерживает создание, редактирование, удаление и фильтрацию заметок.  
Все данные сохраняются в **JSON**-файл и загружаются при следующем запуске.  
Реализован **стек отмены (Undo)** — любое действие можно откатить.

---

## Архитектура (MVC + ООП)

```
notebook-organizer/
├── main.py                              ← Точка входа
├── data/
│   └── notes.json                       ← Хранилище заметок (JSON)
├── src/
│   ├── models/
│   │   ├── note.py                      ← Базовый класс Note
│   │   ├── text\\\_note.py                 ← TextNote (полиморфизм)
│   │   ├── voice\\\_note.py                ← VoiceNote (полиморфизм)
│   │   └── note\\\_factory.py              ← Фабричный метод
│   ├── repository/
│   │   └── note\\\_repository.py           ← Чтение/запись JSON
│   ├── service/
│   │   └── note\\\_service.py              ← Бизнес-логика, CRUD, фильтры, Undo
│   ├── controller/
│   │   └── note\\\_controller.py           ← Обработка ввода (MVC Controller)
│   ├── view/
│   │   └── console\\\_view.py              ← Весь вывод в консоль (MVC View)
│   └── utils/
│       ├── undo\\\_stack.py                ← Стек (LIFO) для отмены действий
│       └── validator.py                 ← Валидация ввода
└── tests/
    └── test\\\_all.py                      ← 78 тестов (unittest)
```

### Применённые паттерны

|Паттерн|Где используется|
|-|-|
|**MVC**|Model (`models/`), View (`console\\\_view`), Controller (`note\\\_controller`)|
|**Factory Method**|`NoteFactory.create()` — создаёт нужный подкласс|
|**Stack (LIFO)**|`UndoStack` — хранит снимки состояния перед каждым изменением|
|**Repository**|`NoteRepository` — изолирует всю работу с файлом|
|**Полиморфизм**|`TextNote` и `VoiceNote` переопределяют `display()` и `to\\\_dict()`|

## Возможности

* ✅ **CRUD** — создание, просмотр, обновление, удаление заметок
* ✅ **Два типа заметок** — `TextNote` (с категорией) и `VoiceNote` (с длительностью и транскрипцией)
* ✅ **Фильтрация по тегу** — без учёта регистра
* ✅ **Фильтрация по дате** — задаётся диапазон от/до
* ✅ **Полнотекстовый поиск** — по заголовку, тексту и тегам
* ✅ **Undo** — отмена последнего действия (глубина стека: 50)
* ✅ **JSON** — автосохранение после каждого изменения, загрузка при старте
* ✅ **Валидация** — информативные сообщения об ошибках для каждого поля

## Примеры использования

### Главное меню

```
══════════════════════════════════════════════════
  📓  Notebook Organizer
══════════════════════════════════════════════════
──────────────────────────────────────────────────
  \\\[1] Список всех заметок
  \\\[2] Добавить TextNote
  \\\[3] Добавить VoiceNote
  \\\[4] Просмотреть заметку
  \\\[5] Изменить заметку
  \\\[6] Удалить заметку
  \\\[7] Фильтр по тегу
  \\\[8] Фильтр по дате
  \\\[9] Поиск
  \\\[U] Отменить действие (Undo)
  \\\[Q] Выход
──────────────────────────────────────────────────
```

### Создание TextNote

```
  Выберите пункт: 2
  Заголовок: Встреча с командой
  Текст: Обсудить бюджет на Q4
  Теги (через запятую, необязательно): работа, встреча
  Категория (необязательно, по умолчанию: general): meetings
✔  TextNote создана: a1b2c3d4e5f6
```

### Создание VoiceNote

```
  Выберите пункт: 3
  Заголовок: Идея на ходу
  Описание: Новый подход к API
  Теги (через запятую, необязательно): dev, идеи
  Длительность в секундах (необязательно, по умолчанию 0): 75
  Транскрипция (необязательно): Рассмотреть REST вместо gRPC
✔  VoiceNote создана: f6e5d4c3b2a1
```

### Фильтр по тегу

```
  Выберите пункт: 7
  Тег для фильтрации: работа
ℹ  Заметки с тегом "работа":
  1. \\\[a1b2c3d4e5f6] \\\[TEXT] Встреча с командой (2024-05-21)
```

### Отмена действия

```
  Выберите пункт: U
✔  Отменено. Заметок восстановлено: 2
```
